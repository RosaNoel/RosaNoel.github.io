Breast cancer Wisconsin dataset¶

Executive summary¶

This project is based on the Wisconsin Breast cancer database. I used the version published on Kaggle by UCI Machine Leraning. The dataset is available under https://www.kaggle.com/uciml/breast-cancer-wisconsin-data.

The objectives of this analysis are to explore classification algorithms, and to demonstrate my python programming skills.

I will fist explore the data, both analytically and visually, then pre-process them. Next, I will calibrate and run a number of classification algorithms:

  1. Naive Bayes
  2. Logistic regression
  3. Support vector machine
  4. Basic classification tree (CART)
  5. Boosted tree (Ada Boost)
  6. Random Forest
  7. Gradient boosting (XGBoost)

Performance measurement strategy¶

A comonly used performance metric is the classification accuarcy, defined as the percentage of observation correctly classified. A naive algorithm that would classify every observation as benigh would achieve a 63% accuracy on this dataset.

Because of the nature of this dataset, we will focus on the ability of the algorithms to detect malignant cells, with a trade-off for mis-diagnosis of non-cancerous patients.

We are more ready to accept that, at the cost of a stressful experience, a patient would be sent to additional sreening before being cleared for breat cancer (False Positive), than to accept the risk that a patient would not receive a treatement for breast cancer when needed (False Negative)

For this strategy, we will compare the different classification algorithms based on the following metrics:

Sensitivity / Recall

This is the algorithm ability to detect positives when the result is actually positive. More formally:

$Sensitivity = \frac{True \ Positives}{True \ Positives \ + \ False \ Negatives} = \frac{Number \ of \ malignant \ cells \ detected \ as \ malignant}{Total \ number \ of \ malignant \ cells}$

F1 Score

This is the balance between precision (percentage of true positives among results predicted as positive) and recall. weighted harmonic mean of the precision and recall

$F1 = 2 \ \times \frac{precision \ \times \ recall}{precision \ + \ recall}$

#create my own confusion matrix (I find the default one too basic)
def ConfusionMatrix(ytrue, ypred):
    matrix = pd.crosstab(ytrue, ypred, rownames=['True'], colnames=['Predicted'], margins=True)
    return matrix

#define a function that will retrieve cross val results in grid search
def CrossValResult(nfold,grid):
    b=np.zeros(nfold)#init output array
    for i in range(nfold):
        b[i]=grid.cv_results_['split'+ str(i) +'_test_score'][grid.best_index_]
    return b
