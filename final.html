Breast cancer Wisconsin dataset - Classification¶

Author : Rosalie Noel

Executive summary¶

1 in 8 women in New South Wales will develop breast cancer in their lifetime*. An early diagnosis and prompt treatement is critical for the patient's survival outlook. The 5-year relative survival rate for women with stage 0 or stage I breast cancer is close to 100%*, while this rate drops to around 22% when the cancer has spread to other parts of the body (stage IV).

This project is based on the Breast Cancer Wisconsin dataset, published by UCI Machine learning. The dataset contains information extracted from digitized image of breast mass cell nuclei, along with the associated diagnosis (Benign / Malignant). With 569 samples, it is a pretty small dataset. It is relatively balanced with 357 benign and 212 malignant observations.

This was one of my first projects in Python. It gave me the opportunity to explore the implementation and tuning of supervised classification algorithms in this language.

I started by exploring the data, both analytically and visually, before implementing a number of classification algorithms (Logistic regression, Tree classifier, Random Forest, XGBoost, ...). The best performer among those base models, a simple classification tree, achieved a recall score of 96.4% (3.6% of the samples labelled as Benign by the classifier are actually malignant).

In a second iteration, I ensembled the 3 best performers with majority vote classifier. This improved model was able to detect all the malignant cases in the sample (100% recall). It achieved a global accuracy score of 98%, as 7% of the samples labeled as malignant by the model were actually benign.

Preparation of the predictive modelling¶

Performance measurement strategy¶

A commonly used performance metric is the classification accuracy, defined as the percentage of observation correctly classified. A naive algorithm that would classify every observation as benign, would achieve a 63% accuracy on this dataset, which is the proportion of benign observations.

This is not the measure that we will adopt for this dataset, as we want to focus on the detection of all malignant cells. We are ready to accept (False Positive), i.e., at the cost of a stressful experience, a patient would be tagged as positive, sent to additional screening and then cleared for breast cancer. But we absolutely want to avoid (False Negative), where a patient is diagnosed with breast cancer and therefore is not receiving appropriate treatement.

With this constraint in mind, we use the following metrics to assess the quality of the classifiers:

Sensitivity / Recall

This is the models's ability to classify as positive when the result is actually positive. More formally:

$Sensitivity = \frac{True \ Positives}{True \ Positives \ + \ False \ Negatives} = \frac{Number \ of \ malignant \ cells \ detected \ as \ malignant}{Total \ number \ of \ malignant \ cells}$

F1 Score

This is the balance between precision (percentage of true positives among results predicted as positive) and recall, expressed as a weighted harmonic mean of both quantities:

$F1 = 2 \ \times \frac{precision \ \times \ recall}{precision \ + \ recall}$

def MyConfusionMatrix(ytrue, ypred):
    matrix = pd.crosstab(ytrue, ypred, rownames=['True'], colnames=['Predicted'], margins=True)
    return matrix

#define a function that will retrieve cross val results in grid search
def CrossValResult(nfold,grid):
    b=np.zeros(nfold)#init output array
    for i in range(nfold):
        b[i]=grid.cv_results_['split'+ str(i) +'_test_score'][grid.best_index_]
    return b

Our objective was to build a model that would detect all malignant cases. The majority vote classifier, based on a Logistic regression, Tree classifier and Random Forest, achieves a 100% recall on the validation set. That means that all the malignant cells are now detected by the classifier.
